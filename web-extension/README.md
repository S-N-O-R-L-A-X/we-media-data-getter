# 百度贴吧数据提取器 - Chrome 扩展

一个简单的 Chrome 浏览器扩展，用于从百度贴吧创作平台自动提取视频数据。

## 功能特点

- 📊 **当前页提取**：一键提取当前页面的视频数据
- 🔄 **自动抓取**：支持自动遍历多页进行批量抓取
- 📅 **日期筛选**：按指定日期范围智能筛选数据
- 💾 **CSV 导出**：导出数据为 CSV 格式，可直接用 Excel 打开
- ⚙️ **可配置**：支持自定义抓取页数、延迟时间等参数

## 提取的数据字段

| 字段 | 说明 |
|------|------|
| 发布日期 | 视频的发布时间 |
| 视频标题 | 视频的名称 |
| 视频链接 | 视频的 URL 地址 |
| 浏览数 | 视频的播放次数 |
| 点赞数 | 视频的点赞数量 |
| 收藏数 | 视频的收藏数量 |
| 回复数 | 视频的评论数量 |
| 分享数 | 视频的分享数量 |

## 安装步骤

### 1. 下载扩展文件

确保 `web-extension` 目录下包含以下文件：

```
web-extension/
├── manifest.json       # 扩展配置文件
├── content-script.js   # 内容脚本
├── background.js       # 后台服务 worker
├── popup.html          # 弹窗页面
├── popup.js            # 弹窗脚本
├── popup.css           # 弹窗样式
├── welcome.html        # 欢迎页面
└── icons/              # 图标目录
    ├── icon-16.svg
    ├── icon-48.svg
    └── icon-128.svg
```

### 2. 加载到 Chrome

1. 打开 Chrome 浏览器，访问 `chrome://extensions/`
2. 开启右上角的 **"开发者模式"**
3. 点击 **"加载已解压的扩展程序"** 按钮
4. 选择 `web-extension` 文件夹
5. 扩展加载成功后会在扩展列表中看到 "Tieba Data Extractor"

## 使用方法

### 基本使用流程

1. **打开百度贴吧创作平台**
   - 访问：https://tieba.baidu.com/home/creative/work

2. **点击扩展图标**
   - 在浏览器右上角点击扩展图标，打开数据提取器面板

3. **开始提取数据**
   - 点击「提取当前页数据」立即提取当前页数据
   - 或输入目标页数后点击「开始自动抓取」进行批量抓取

4. **导出 CSV 文件**
   - 抓取完成后，点击「导出为 CSV 文件」按钮
   - 文件将自动下载到系统默认下载目录

### 详细操作步骤

#### 提取当前页数据

1. 确保在正确的页面（百度贴吧创作平台）
2. 点击「提取当前页数据」按钮
3. 系统会自动提取当前页的视频数据并显示统计信息

#### 自动抓取多页

1. 在「目标页数」输入框中设置需要抓取的页数（默认为 42 页）
2. 点击「开始自动抓取」按钮
3. 系统会自动翻页并提取每一页的数据
4. 如需中途停止，点击「停止抓取」按钮

#### 导出数据

1. 点击「导出为 CSV 文件」按钮
2. 数据会按日期降序排列并导出
3. 文件名格式：`tieba_videos_YYYY-MM-DD.csv`

#### 配置参数

在设置区域可以调整以下参数：

| 参数 | 说明 | 默认值 |
|------|------|--------|
| 日期截止点 | 只提取此日期之后的数据 | 2026-05-25 |
| 默认最大页数 | 自动抓取时的最大页数限制 | 42 |
| 翻页延迟 | 每次翻页前的等待时间（秒） | 3 |

修改后点击「保存设置」即可生效。

## 注意事项

1. **页面要求**：扩展只在 `https://tieba.baidu.com/home/creative/work` 页面生效
2. **数据准确性**：数据来源于页面的 Vue 组件 props，可能会因页面更新而变化
3. **抓取速度**：建议不要设置过小的翻页延迟，避免被网站限流
4. **Cookie 要求**：需要有登录状态才能查看完整的视频数据

## 故障排除

### 无法提取数据

- 确认是否在正确的页面：`https://tieba.baidu.com/home/creative/work`
- 尝试刷新页面后重新打开扩展
- 检查浏览器控制台是否有错误信息

### 自动抓取失败

- 增加翻页延迟时间（建议设置为 5-10 秒）
- 检查网络连接是否正常
- 确认网页是否完整加载（部分内容为懒加载）

### CSV 导出乱码

- 导出的 CSV 文件包含 BOM 头，建议在 Excel 中使用 UTF-8 编码打开
- 或使用文本编辑器打开查看

## 开发指南

### 项目结构

```
web-extension/
├── manifest.json       # Manifest V3 配置文件
├── content-script.js   # 运行在目标页面的脚本
├── background.js       # 后台服务 Worker（消息处理、通知等）
├── popup.html/js/css   # 扩展弹窗界面
├── welcome.html        # 首次安装后的欢迎页面
└── icons/              # 扩展图标
```

### 添加新扩展

如需添加新的数据提取功能：

1. 修改 `manifest.json` 中的 `content_scripts` 添加新的匹配规则
2. 创建新的内容脚本文件
3. 在 `popup.html` 中添加相应的 UI 元素
4. 编写对应的交互逻辑

### 调试方法

1. 在 `chrome://extensions/` 页面点击扩展的 "Service Worker" 链接
2. 在弹出的 DevTools 中可以看到后台日志
3. 在目标页面打开 DevTools 可以看到 content script 日志
